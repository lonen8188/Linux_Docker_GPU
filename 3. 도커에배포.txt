--------------도커 결론- ---------------------
결론

👉 소스코드만 수정했으면 docker compose up만 하면 됨 (O)
👉 단, 조건 있음

🔍 지금 너 환경 기준
volumes:
  - ./flask-app:/app

👉 이거 때문에:

👉 코드가 컨테이너에 실시간 반영됨

✅ 그래서 경우별 정리
1️⃣ Python 코드만 수정 (Flask, YOLO 등)
docker compose up -d

👉 또는 그냥 놔둬도 됨 (이미 반영됨)

✔ 이유:

볼륨 마운트 → 코드 즉시 반영

2️⃣ requirements.txt 수정 (pip 추가)
docker compose build
docker compose up -d

👉 이유:

pip는 이미지 안에 설치됨 → 재빌드 필요

3️⃣ Dockerfile 수정
docker compose build --no-cache
docker compose up -d
4️⃣ GPU / 환경 설정 바꿈
docker compose down
docker compose up -d
🚨 중요한 포인트
❗ Flask 재시작 필요할 수도 있음

코드만 바꿨는데 반영 안 되면:

docker compose restart flask
🔥 추천 개발 루틴
# 코드 수정 후
docker compose restart flask

👉 이게 제일 빠름

🎯 한 줄 정리

👉 코드만 수정 = restart
👉 라이브러리 변경 = build

💣 너한테 딱 맞는 공식
코드 → restart
pip → build
Dockerfile → no-cache build
